单文件配置 本地 API proxysss DNS-01

1. 配置原则

  • 唯一主配置：provider 现在只写 server.yaml；旧的 agent-config.json 仅作为一次性迁移输入。
  • Agent inspectability：本地接口全部是 JSON 或纯文本/YAML，避免多文件 include 与隐式状态。
  • Gateway as code：proxysss 网关计划可直接生成 YAML，也可通过 admin API 一键注册路由。

3. 本地 Agent API

以下接口仅允许 127.0.0.1 / ::1 访问，适合本机 Agent/脚本编排。

接口输出用途
GET /api/v1/local/configJSON读取当前 server.yaml 生效配置
GET /api/v1/local/protocolsJSON协议能力矩阵
GET /api/v1/local/generate.shShell节点安装脚本
GET /api/v1/local/subscribe/convertYAML/JSON/TXT订阅转换
GET /api/v1/local/gateway/proxysss.yamlYAML生成 proxysss 网关 + DNS-01 ACME 配置
POST /api/v1/local/gateway/registerJSON向 proxysss admin API 自动注册 provider 路由

4. CLI 对应面

go run . config get
go run . config set --proxysss-enabled=true --dns-enabled=true \
  --dns-provider=cloudflare --dns-api-token=cf-token \
  --dns-email=ops@example.com --dns-base-domain=example.com

go run . gateway-plan
go run . gateway-register

go run . generate --format sh --node-ip edge.example.com \
  --enable-acme=true --acme-domain=edge.example.com \
  --protocols=singbox-nekotls,mihomo-nekotls

5. proxysss 自动化流

  • 步骤 1：在 server.yaml 打开 proxysss.enabled 与 dns_automation.enabled。
  • 步骤 2：用 GET /api/v1/local/gateway/proxysss.yaml 生成完整 gateway YAML。该 YAML 已包含 acme_managed + dns01、监控、watchdog、以及 provider 面板域名路由。
  • 步骤 3：将该 YAML 交给 proxysss：proxysss -config ./proxysss.yaml check-config，再启动或热重载。
  • 步骤 4：如果 proxysss 已运行并开启了 admin.enable_write_ops，直接调用 POST /api/v1/local/gateway/register 自动 upsert 路由。
curl -s http://127.0.0.1:8080/api/v1/local/gateway/proxysss.yaml -o proxysss.yaml
proxysss -config ./proxysss.yaml check-config
proxysss -config ./proxysss.yaml start

curl -X POST http://127.0.0.1:8080/api/v1/local/gateway/register

6. 验证闭环

  • 单测：YAML 迁移、gateway YAML 生成、proxysss admin route register
  • 语法：bash -n 校验外层安装器与内嵌 vpn233-node CLI
  • 端到端：scripts/verify.sh / scripts/verify.ps1 会额外校验 gateway YAML 含 challenge: dns01 / provider: cloudflare
  • CI：GitHub Actions 在 Ubuntu / Windows 运行完整测试与 verify 脚本，发布后可用 gh run watch 验证
