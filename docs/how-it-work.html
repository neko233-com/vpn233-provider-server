1. 整体协作流

管理台 登录 -> 勾选协议 -> 生成脚本

2. 订阅服务器校验机制

每次订阅服务启动或更新前可执行验证，确认 provider 已具备基础能力。

curl -s http://127.0.0.1:8080/api/v1/subscribe/verify?token=YOUR_VERIFY_TOKEN | jq .

5. 协议参数透传技术原理

所有协议都走统一入口：Provider 在生成阶段为每个出站/入站计算 ProtocolPort.Options，再按核心进行深度合并。

sing-box 分支

策略返回一份标准入站 Map 后，执行 deepMergeStringAnyMaps(inbound, options)，支持层级覆盖（例如 tag_meta）。

mihomo 分支

策略返回 YAML Proxy 行后先反序列化为对象，再用同样的深合并覆盖，最后重排为 proxies[] 片段。

统一参数入口

CLI/Query 使用 protocol_options 或 protocol_options_b64，格式统一为： {"协议id":{"key":value}}。

// singbox: /api/v1/generate?protocol_options={"singbox-vless":{"tag":"vip","tag_meta":{"tier":"gold"}}}
// mihomo: {"mihomo-vless":{"network":"tcp","ws-opts":{"path":"/edge","headers":{"Host":"edge.example.com"}}}
