超高性能 + 安全加固

生成的节点脚本自动写入并应用以下调优（均可开关）。

⚡ 性能调优 /etc/sysctl.d/99-vpn233.conf

  • 拥塞控制：BBR + fq（可切 cubic 等），自动 modprobe tcp_bbr
  • 大缓冲：rmem/wmem 128MB，UDP min 缓冲利好 QUIC/Hysteria2/TUIC
  • 高并发：somaxconn / syn_backlog 65535，netdev_backlog 250000，tw_reuse
  • TFO：tcp_fastopen=3；MPTCP：可选 net.mptcp.enabled
  • 句柄：file-max 2097152，limits.d + systemd DefaultLimitNOFILE
  • 服务加固：LimitNOFILE/NPROC + 网络能力 capability

🛡️ 安全 / 运维

  • sysctl 加固：syncookies、rp_filter、禁重定向/源路由、kptr/dmesg 限制
  • fail2ban：sshd jail 防爆破（best-effort，不中断）
  • ACME：acme.sh standalone 签发 + 自动续期，失败回退自签
  • 看门狗：systemd timer 每 2 分钟自愈拉起异常服务
  • 日志轮转：logrotate 保留 7 天，copytruncate
  • 一键运维：backup / restore / update / uninstall / speedtest / doctor

NekoTLS · neko233 旗舰协议

把 uTLS 指纹 + ECH + Reality + AnyTLS padding 统一封装成单一 type。

设计

  • 外层线路：AnyTLS 兼容流（TLS 1.3 + padding + 多路复用）
  • 服务端：stock sing-box 原生 anytls 入站即可落地
  • 域名模式：ECH 隐藏真实 SNI，校验正式证书
  • 免域名模式：Reality 借壳，sni 指向借壳站点
  • 客户端：mihomo fork 原生 type: nekotls

为什么需要 fork

stock 核心无法解析未知 type:，且官方 mihomo 不支持 AnyTLS + Reality 组合、也不能把指纹/ECH/Reality 收敛进单一出站。 fork 仅改动出站解析 + 一个 outbound，伪装能力全部复用 mihomo 既有的 anytls / ech / reality / utls 组件。

契约 NekoTLSOption / DecodeNekoTLSOption 由 provider 与 fork 共享；CI 用同一解码器校验所有生成产物，保证可加载。 详见 mihomo-fork/PLAN.md。

API 速查

登录获取 Bearer token；回环地址可走 /api/v1/local/* 免登录接口。

主要接口

  • POST /api/v1/login
  • GET /api/v1/health · /protocols · /config
  • POST /api/v1/generate · GET /generate.sh|.ps1
  • GET /api/v1/subscribe/verify
  • GET /api/v1/subscribe/convert
  • GET /repo/status · POST /repo/sync

generate 性能/安全参数

  • enable_bbr · tcp_congestion · enable_tcp_fastopen · enable_mptcp
  • conn_limit（默认 1048576）
  • enable_hardening · enable_fail2ban
  • enable_logrotate · enable_watchdog
  • enable_acme · acme_domain · acme_email
  • selected_protocols · node_ip · port_base
